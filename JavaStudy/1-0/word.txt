サーバーソフト
	Apache : アパッチ。世界的に使用されるWebサーバソフトの一つ。高い性能と優れた動作性を持つ。無料で使用可能なので、費用をかけずにWebサーバーを使うことができる。
	nginx : Apacheに次いで2番目に人気のサーバソフト。Apacheと比べて動作が速く、複数のアクセスに対して負荷がかかりにくい。動画を扱うWebサイトではApacheに劣る。
	IIS : Microsoft Internet Information Servicesの略。マイクロソフトが開発したWebサーバーソフト。

SQL
	SQL : Structured Query Languageの略。データベースを操作するための言語。
		DB2 : IBMの開発したデータベース管理システム。初めてSQLを採用したデータベース。高速で処理するXML機能を持つ。
		MySQL : オープンソースで無料のデータベース管理システムで、さまざまなOSに対応している。WordPressと相性がいい。大規模なアプリケーションまで対応可能。複雑な処理は不向き。
			Maria DB : MySQLから派生したレーショナルデータベース。MySQLをベースに新機能追加やソースコードが改善されている。
		Oracle DB : Oracle社が開発した世界で高いシェアを持つデータベース。C言語で成り立っており、他のプラットフォームへ移植しやすい。
		PostgreSQL : オープンソースで無料で利用することができる。多くのプラットフォームに対応可能。複雑な処理は対応可能。
		SQLite : 簡易的に利用することが可能。「自己完結型システム」と呼ばれており、ライブラリとして手軽に使用できる。
		SQL Server : Microsoft社が開発したデータベースシステム。企業で利用されることが多い。
	CRUD : 「Create, Read, Update, Delete」（クラッド）の略。データを操作するときに必要な機能の表現。
	トランザクション（一連の処理）
		コミット : データベースの状態を処理結果によって更新すること。確定処理。
		ロールバック : 処理に失敗した場合に、トランザクション開始前の状態に戻すこと。
	排他制御（排他ロック） : 共有可能な資源に同時にアクセスしても問題なく動作させる制御。
		楽観ロック :ある処理でデータを更新する前後において、別の処理によってデータが更新されていないかを確認する。他のプログラムによる更新が確認された場合ロックがかかる。
		悲観ロック : あるデータがアクセスされているときに、データをロックし他のアクセスを禁止すること。
	実行計画 : SQLが実行するときの手順の組み合わせ。
	チューニング : パフォーマンスが悪い時にときに行う処理。非効率なSQLの改善など。コストの高い処理を改善するなど。処理時間が許容範囲になるまで改善する。
パフォーマンス : 
	スロークエリ : SQLの実行時間が指定した時間よりも長くかかってしまった場合に出力される。パフォーマンス向上の妨げの原因を発見することができる。
	クエリログ : SQLで実行された全ての記録。

	DBツール
		A5:SQL Mk-2 : データベースの開発ツール/ER図ツール。高機能、軽量、分かりやすい。
		pgAdmin : データベースサーバをグラフィカルに表示・操作できるもの。Webブラウザから動作するためOSは不問。
		DBeaver : データベースツール。A5:SQL Mk-2とは違い、対応しているOSが幅広い。
		MySQL Workbench : MySQLをGUIで分かりやすく操作するツール。
		Sequel Pro : Mac用のSQLデータ管理ソフト。
		TeamSQL : データベースの種類に関係なく横断的に使えるSQLクライアント。よって、インストールするDBクライアントが少なく済む。

ネットワーク
	LAN : 「Local Area Network」の略。限定された範囲でのネットワーク。クローズドなネットワーク。
	WAN : 「Wide Area Network」の略。広範囲のネットワーク。インターネット。
	クライアント : サービスを利用する側。
	サーバー : サービスを提供する側。
各種プロトコル : コンピューターでデーターをやり取りをするための定められた手順や規約。
	HTTP : 「Hypertext Transfer Protocol」の略。WebサーバとWebブラウザの間、Web情報をやり取りをするためのプロトコル。
	FTP : 「File Transfer Protocol」の略。サーバとクライアントの間でファイルの転送を行う際に必要となる通信プロトコル。
	HTTPS : 「Hypertext Transfer Protocol Secure」の略。HTTPと比べて暗号化が加えられている。
	SSL : 「Secure Sockets Layer」の略。インターネット上におけるウェブブラウザとウェブサーバの間でのデータ通信を暗号化し、送受信させる仕組み。
	SSH : 「Secure Shell」の略。ネットワークに接続された機器を遠隔操作し、管理するための手段。
	ポート、ポート番号 : プログラム単位に割りられる番号。通信を行なっているアプリケーションを識別するための番号。
	HTTP、FTP、SSHの各ポート番号
	ファイアウォール
	TCP : 「Transmission Control Protocol」の略。「送ったデータが相手に届いているか確認しながら通信する方法」。
IPアドレス
	IPアドレス : コンピュータやスマートフォンに割り当てられるインターネット上の識別番号。
		グローバルIPアドレス : 全世界の中で識別できるように割り振られたIPアドレス。世界に一つ。
		プライベートIPアドレス : ローカルネット上で割り振られるIPアドレス。
ドメイン
	ドメイン : インターネット上でWebサイトを識別するための情報。
	DNS : 「Domain Name System」の略。ドメイン名とIPを紐づけるシステム。人間が見て分かりやすい「ドメイン名」をインターネット上の通信で使われるIPアドレスに変換する。
	VPN : 「Virtual Private Network」の略。ネットワーク上に仮想の専用線を構築し、安全かつ安定した通信を実現したネットワーク環境。社内のネットワークの構築などに使われる。
	ブリッジ : ネットワークにおいて端末同士を接続するための装置。
	プロキシサーバー : インタネットに接続できないコンピュータに代わり、インターネットに接続しWebサイトへのアクセスなどを行うサーバー。
電子メール
	SMTP : 「Simple Mail Transfer Protocol」の略。メールを送信するための通信プロトコル。
	POP : 「Post Office Protocol」の略。メールを受信するための通信プロトコル。
	IMAP : 「Internet Message Access Protcol」の略。サーバーから受信したメールをサーバに残す。ユーザーのメールはサーバーが管理する。
セッションとクッキー : 別のページに遷移しても必要となるデータを保存するしくみ。
	セッション : Webサーバーに保存される。安全性が高い。
	クッキー : ブラウザに保存される。
